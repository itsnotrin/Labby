<objective>
Fix timer retain cycles in all four ViewModels that use Timer.scheduledTimer with implicit self capture.

Purpose: Timers hold a strong reference to self through the closure, preventing ViewModel deallocation when the view disappears. This causes memory leaks where ViewModels (and their associated network clients) accumulate over repeated navigation.

Output: Four timer closures updated to use [weak self] capture lists with guard-let early return.
</objective>
