<objective>
Resolve race conditions on shared mutable state in JellyfinClient, PiHoleClient, and ProxmoxClient by converting them from `final class` to `actor`.

Purpose: All three service clients are `final class` with unprotected mutable instance properties accessed from concurrent async contexts. Multiple async calls can race on token read/write (Jellyfin), session read/write (PiHole), and static dictionary mutation (Proxmox). Converting to `actor` provides Swift-native data isolation with zero manual locking.

Output: Three client files converted from `final class` to `actor` with all call sites updated for actor isolation.
</objective>

Convert JellyfinClient to use actor isolation for thread-safe access to cached auth state.

**Step 1: Change class declaration (line 10):**
```swift
// FROM:
final class JellyfinClient: ServiceClient {
// TO:
actor JellyfinClient: ServiceClient {
```

**Step 2: Handle `lazy var session` (line 18):**
Actors do not support `lazy var`. Convert to a computed property backed by a stored optional, or use `nonisolated(unsafe) lazy var`. The simplest approach: change to a stored `nonisolated let` initialized in `init`, OR keep as `nonisolated(unsafe) lazy var` since URLSession is thread-safe internally.

Recommended approach — use `nonisolated(unsafe) lazy var`:
```swift
nonisolated(unsafe) lazy var session: URLSession = {
    // ... existing implementation unchanged
}()
```
This is safe because URLSession is itself thread-safe. The `nonisolated(unsafe)` annotation tells the compiler we've verified safety manually.

**Step 3: Handle `nonisolated` for immutable properties:**
The `config` property is a `let` so it's already implicitly `nonisolated` in actors. The `tokenExpiryDuration` is also `let` — same. The `deviceId` (lazy var) needs the same treatment as `session` — `nonisolated(unsafe) lazy var`.

**Step 4: Fix compilation errors:**
After converting to actor, the compiler will flag:
- Any synchronous external access to actor properties → these should already be async (all client methods are async)
- `Sendable` conformance issues → check if ServiceClient protocol requires class conformance. If ServiceClient has `: AnyObject` or `: class`, it may need adjustment. If it's just a protocol, actors can conform directly.
- If `testConnection()` is called from `@MainActor` ViewModels, those calls should already use `await` since all client methods are `async throws`

**Step 5: Build and fix any remaining compiler errors iteratively.**

Key insight: Since all methods on JellyfinClient are already `async throws`, callers already use `await` — the actor conversion should require minimal call-site changes. The main risk is the `lazy var session` and any synchronous property access from outside.

**Step 2: Handle static var netSnapshots (line 11):**
This is the MOST CRITICAL race condition — static mutable state shared across all ProxmoxClient instances. Converting to actor isolates instance properties but NOT static properties. For the static dictionary, two approaches:

**Recommended approach — move to a dedicated global actor or use a simple actor wrapper:**
```swift
private actor NetSnapshotStore {
    static let shared = NetSnapshotStore()
    private var snapshots: [UUID: (ts: TimeInterval, inBytes: Int64, outBytes: Int64)] = [:]

    func get(_ id: UUID) -> (ts: TimeInterval, inBytes: Int64, outBytes: Int64)? {
        snapshots[id]
    }

    func set(_ id: UUID, value: (ts: TimeInterval, inBytes: Int64, outBytes: Int64)) {
        snapshots[id] = value
    }
}
```

Then replace all `ProxmoxClient.netSnapshots[config.id]` reads with `await NetSnapshotStore.shared.get(config.id)` and writes with `await NetSnapshotStore.shared.set(config.id, value: ...)`.

**Alternative simpler approach:** Use `nonisolated(unsafe) private static var netSnapshots` if the risk is deemed low enough — but this does NOT fix the race, only silences the compiler. The dedicated actor is the correct fix.

Use the dedicated `NetSnapshotStore` actor approach.

**Step 3: Handle `lazy var session`:**
Same as Tasks 1 and 2 — `nonisolated(unsafe) lazy var`.

**Step 4: Instance cache properties (lines 15-22):**
