**isUsingCachedData (~line 254 in ProxmoxViewModel.refresh):**

Current logic: `self.isUsingCachedData = !dataChanged && beforeNodes > 0` — relies on count comparison which is unreliable if the server happens to have the same number of items.

Fix: The ProxmoxClient actor (Phase 2) already has timestamp fields (`nodesTimestamp`, `vmsTimestamp`, `storageTimestamp`) for cache freshness. Instead of comparing counts before/after, capture the timestamps before the fetch and compare after:

1. Before `fetchDetailedData()`, capture `let beforeTimestamp = await client.vmsTimestamp` (or use a combined approach)
2. After `fetchDetailedData()`, check: `let afterTimestamp = await client.vmsTimestamp`
3. Set: `self.isUsingCachedData = (beforeTimestamp != nil && beforeTimestamp == afterTimestamp)`

If timestamps match, data came from cache. If timestamp changed, fresh data was fetched. This is reliable regardless of count.

Note: If the client timestamps are not accessible (private), use a simpler approach: add a `var lastFetchWasCached: Bool` property to ProxmoxClient that gets set to `true` when returning cached data and `false` when fetching fresh data from the API. Then read it after the fetch.

**mergeVMs (TWO copies — ~line 351 in ProxmoxViewModel and ~line 925 in ProxmoxDetailViewModelLarge):**

Current logic: starts dict from existing VMs, then adds/updates from new — never removes stale entries.

Fix BOTH copies with the same approach — reverse the merge direction:

```swift
private func mergeVMs(existing: [ProxmoxVM], new: [ProxmoxVM]) -> [ProxmoxVM] {
    // Build lookup from existing VMs for identity preservation
    let existingDict = Dictionary(uniqueKeysWithValues: existing.map { ($0.vmid, $0) })

    // Start from new VMs only — this naturally excludes stale entries
    let merged = new.map { newVM -> ProxmoxVM in
        if let existingVM = existingDict[newVM.vmid], existingVM == newVM {
            return existingVM  // Preserve identity if unchanged
        }
        return newVM  // Use new data
    }

    return merged.sorted { $0.vmid < $1.vmid }
}
```

Key change: iterate over `new` (not `existing`), so VMs that no longer exist on the server are naturally excluded. Preserve object identity from `existing` only when the data hasn't changed (to avoid unnecessary SwiftUI re-renders).
