# Contributing to Labby

Thank you for considering contributing to Labby! We welcome contributions in code, documentation, and testing. Please follow the guidelines below to ensure consistency and maintain the high quality of the project.

## General Guidelines

- **Consistency:** Adhere to the existing code style and conventions used throughout Labby.
- **Testing:** Ensure that every change is thoroughly tested on an iOS device or simulator. No untested changes should be merged.
- **Documentation:** Update or add documentation as necessary, especially when introducing new features or making significant changes.
- **Commit Messages:** Use clear and descriptive commit messages that explain the purpose and context of your changes.

## How to Contribute

1. **Fork the Repository:** Start by forking the Labby repository, then clone your fork locally.
2. **Create a Branch:** Create a new branch for your feature or bug fix:
   ```
   git checkout -b feature/your-feature-name
   ```
3. **Implement Your Changes:** Make your changes while following the projectâ€™s coding standards and structure.
4. **Test Your Changes:** Verify that your changes work correctly and do not introduce regressions.
5. **Commit Your Changes:** Write concise commit messages, and ensure that each commit is meaningful.
6. **Submit a Pull Request:** Push your branch to your fork and open a pull request. Include a clear description of your changes and reference any related issues.

## Code Style

- Follow the established project structure and naming conventions.
- Write clean, maintainable code with appropriate documentation where necessary.
- For major changes, please discuss your approach by opening an issue in the repository before implementing.

## Reporting Issues

If you encounter any bugs or have suggestions for improvements, please create an issue in the repository before submitting a pull request. This helps us manage the project effectively and plan future development.

---

Your contributions make Labby better for everyone. Thank you for your support and happy coding!
